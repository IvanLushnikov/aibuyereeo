# Проверка `/api/ktru-lookup` (локально)

## Этап 1. Запуск dev-сервера

- Команда: `npm run dev`
- Порт: `http://localhost:3000`

## Этап 2. HTTP-запрос

```
curl -X POST http://localhost:3000/api/ktru-lookup \
  -H "Content-Type: application/json" \
  -d '{"productName":"автомобиль"}'
```

## Этап 3. Результат

- Полный JSON сохранён в `landing-app/test-results/ktru-lookup-sample.json`.
- В ответе пришло 8 карточек КТРУ. Пример первых двух:

```
Код: 29.10.20.000-00000013
Наименование: Автомобиль легковой
Обязательные характеристики:
  - Тип двигателя: бензиновый
  - Рабочий объём двигателя: 2000–3000
  - Тип привода: моноприводный / полноприводный
  - Тип коробки передач: механика / автомат / робот / вариатор

Код: 29.10.20.000-00000012
Наименование: Автомобиль легковой
Обязательные характеристики:
  - Тип двигателя: бензиновый
  - Рабочий объём двигателя: 1500–2000
  - Тип привода: моноприводный / полноприводный
  - Тип коробки передач: механика / автомат / робот / вариатор
```

- В каждой карточке также возвращаются дополнительные (необязательные) характеристики и список стандартов (`ТР ТС 018/2011`, `ГОСТ 33670-2015`, `ГОСТ Р 52051-2003` и др.).
