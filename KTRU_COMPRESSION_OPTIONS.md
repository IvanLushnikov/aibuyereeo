# Варианты сокращения формата plain для API КТРУ

## Текущий формат
```
CODE | Название характеристики: Значение1, Значение2 | Другая характеристика: Значение3
```

Пример:
```
26.20.11.110-00000138 | Размер диагонали экрана: 17 | Тип процессора: Intel Core i5, Intel Core i7
```

## Варианты сокращения

### Вариант 1: Без названий характеристик (только значения)
**Формат:** `CODE | Значение1, Значение2 | Значение3`

**Пример:**
```
26.20.11.110-00000138 | 17 | Intel Core i5, Intel Core i7
```

**Экономия:** ~30-50% символов
**Плюсы:** Значительно короче
**Минусы:** Менее понятно без контекста

---

### Вариант 2: Без пробелов, короткие разделители
**Формат:** `CODE|P1:V1,V2|P2:V3`

**Пример:**
```
26.20.11.110-00000138|Д-ль:17|Проц:Intel Core i5,Intel Core i7
```

**Экономия:** ~15-20% символов
**Плюсы:** Читаемо, компактно
**Минусы:** Требует словарь сокращений

---

### Вариант 3: Только первое значение
**Формат:** `CODE | Характеристика: Значение1 | Другая: Значение1`

**Пример:**
```
26.20.11.110-00000138 | Размер диагонали экрана: 17 | Тип процессора: Intel Core i5
```

**Экономия:** ~40-60% при множественных значениях
**Плюсы:** Сохраняет читаемость
**Минусы:** Теряется информация о других вариантах

---

### Вариант 4: Сокращенные названия через словарь
**Формат:** `CODE | Сокр: Значение1, Значение2 | Сокр2: Значение3`

**Пример:**
```
26.20.11.110-00000138 | Д-ль: 17 | Проц: Intel Core i5, Intel Core i7
```

**Экономия:** ~20-30% символов
**Плюсы:** Читаемо, компактно
**Минусы:** Требует поддержки словаря сокращений

---

### Вариант 5: Максимально компактный (только значения)
**Формат:** `CODE|V1,V2|V3`

**Пример:**
```
26.20.11.110-00000138|17|Intel Core i5,Intel Core i7
```

**Экономия:** ~50-70% символов
**Плюсы:** Максимально коротко
**Минусы:** Совсем непонятно без контекста

---

### Вариант 6: Комбинированный (короткие названия + без пробелов)
**Формат:** `CODE|Сокр:V1,V2|Сокр2:V3`

**Пример:**
```
26.20.11.110-00000138|Д-ль:17|Проц:Intel Core i5,Intel Core i7
```

**Экономия:** ~25-35% символов
**Плюсы:** Баланс между читаемостью и компактностью
**Минусы:** Требует словарь сокращений

---

## Рекомендации

1. **Для максимальной экономии контекста:** Вариант 5 (только значения)
2. **Для баланса читаемости и компактности:** Вариант 1 (без названий, но с пробелами) или Вариант 6 (сокращения)
3. **Если важна читаемость:** Вариант 3 (только первое значение)

## Дополнительные оптимизации

- Ограничить количество характеристик (например, только первые 3-5 обязательных)
- Обрезать длинные значения (например, "Intel Core i5" → "i5")
- Использовать коды вместо полных названий характеристик
